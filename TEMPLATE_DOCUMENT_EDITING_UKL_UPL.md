## üìù Instruksi untuk Kandidat

### Latar Belakang
Anda akan menerima dokumen UKL/UPL yang **tidak teratur dan belum lengkap**. Tugas Anda adalah memperbaiki dokumen tersebut mengikuti standar penyusunan dokumen lingkungan hidup yang berlaku.

### Tugas yang Harus Diselesaikan

1. **Rapihkan Dokumen** (25 poin)
   - Perbaiki formatting (font, spacing, alignment)
   - Konsistensi heading hierarchy
   - Tambahkan/perbaiki page numbering
   - Atur margin dan layout profesional

2. **Sesuaikan Kaidah Penyusunan & Penomoran** (25 poin)
   - Perbaiki penomoran bab/sub-bab (1, 1.1, 1.1.1, dst)
   - Beri nomor pada semua tabel dan gambar
   - Update daftar isi
   - Pastikan konsistensi cross-reference

3. **Lengkapi Konten yang Kosong** (30 poin)
   - Isi rumus/formula perhitungan yang hilang
   - Lengkapi data tabel dengan dummy data yang relevan
   - Tambahkan parameter lingkungan standar
   - Isi bagian metodologi yang kosong

4. **Berikan Komentar Teknis** (15 poin)
   - Gunakan fitur "Comment" di Word
   - Identifikasi kesalahan teknis
   - Beri saran perbaikan
   - Tandai bagian yang kritis/perlu review lebih lanjut

5. **Overall Quality** (5 poin)
   - Profesionalisme dokumen
   - Kelayakan untuk diserahkan ke klien
   - Compliance dengan standar UKL/UPL

### Aturan Pengerjaan
- ‚úÖ Gunakan Microsoft Word atau kompatibel (.docx)
- ‚úÖ Boleh menggunakan internet untuk referensi standar/regulasi
- ‚úÖ Simpan dengan nama: `UKL_UPL_FIXED_[NAMA_ANDA].docx`
- ‚ùå Tidak boleh konsultasi dengan orang lain
- ‚è∞ Upload sebelum waktu habis (2 jam)

#### ‚ùå Kesalahan Formatting:
- [ ] Font tidak konsisten (mix Arial, Calibri, Times New Roman)
- [ ] Heading tidak menggunakan Styles
- [ ] Line spacing tidak seragam (ada 1.0, 1.15, 1.5)

#### ‚ùå Kesalahan Penomoran:
- [ ] Penomoran bab salah (Bab 1, BAB II, bab III)

- [ ] Gambar nomor acak (Gambar 1, Figure 2, Gbr. 3)

- [ ] Rumus perhitungan hanya tulisan "[RUMUS DISINI]"

- [ ] Parameter baku mutu tidak ada
- [ ] Metodologi hanya 1 paragraf singkat
- [ ] Data koordinat lokasi kosong

- [ ] Satuan salah (mg/l ditulis mgl)
- [ ] Nama parameter salah ejaan (Total Suspended Solid)


### C. HR/Evaluator Reviews Submission
```
1. Dashboard ‚Üí Test Sessions ‚Üí Filter "Pending Evaluation"
2. Open candidate submission
3. Download:
   - Original template (reference)
   - Candidate submission
   - Answer key (if available)
4. Evaluate using 22 kriteria:
   - Score each criterion (0 to max points)
   - Add evaluator notes
5. Submit evaluation
6. System auto-calculates:
   - Total score
   - Percentage
   - Pass/Fail status
7. Candidate receives result notification
```

## üîß Troubleshooting

### Issue: File upload fails
**Solution:** 
- Check file size < 10MB
- Ensure file extension is .doc or .docx
- Compress images in document if too large

### Issue: Timer doesn't start
**Solution:**
- Timer starts when candidate clicks "Download Template"
- Check TestSession record: started_at should be set
- Verify route: `/candidate/test/{token}/download-template`

### Issue: Can't submit document
**Solution:**
- Ensure time limit not exceeded
- File must be .docx format
- Check network connection
- Verify route: `/candidate/test/{token}/submit-document`

### Issue: Evaluation form doesn't show all criteria
**Solution:**
- Clear browser cache
- Check TestTemplate: evaluation_criteria should be JSON array
- Verify controller: `showEvaluationForm()` method
- Inspect blade template: evaluation form view

## ‚ú® Tips untuk Kandidat

1. **Baca instruksi dengan teliti** - Jangan skip bagian apapun
2. **Manajemen waktu** - Alokasikan:
   - 20 menit: Review dokumen & identifikasi error
   - 60 menit: Perbaikan formatting & penomoran
   - 30 menit: Melengkapi konten & formula
   - 10 menit: QC & upload
3. **Gunakan fitur Word dengan maksimal**:
   - Styles untuk heading
   - Table of Contents auto-generate
   - Insert Caption untuk tabel/gambar
   - Track Changes untuk dokumentasi
   - Comments untuk technical review
4. **Prioritaskan high-value items** - Content completion (30 poin) adalah yang terberat
5. **Double-check sebelum upload** - Pastikan file bisa dibuka dan tidak corrupt

## ‚ú® Tips untuk Evaluator

1. **Gunakan answer key** sebagai baseline
2. **Konsisten dalam penilaian** - Buat rubrik internal jika perlu
3. **Beri nilai parsial** - Jika kandidat usaha tapi belum sempurna, beri poin sebagian
4. **Dokumentasikan alasan** - Tulis notes kenapa dapat poin tertentu
5. **Compare antar kandidat** - Normalisasi scoring untuk fairness
6. **Time-based penalty** (optional) - Kurangi 5 poin jika telat upload
